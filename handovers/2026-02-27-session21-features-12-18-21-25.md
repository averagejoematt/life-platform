## What Was Done

### Feature #12 — Social Connection Scoring (2 MCP tools)
- **`get_social_connection_trend`** — Aggregates `enriched_social_quality` (alone/surface/meaningful/deep) from journal entries. Rolling 7d/30d averages, meaningful-connection streaks, PERMA wellbeing model context, correlations with recovery/HRV/sleep/stress/Body Battery, meaningful-vs-low-social health comparison.

- No new Lambda or DynamoDB partition — reads existing `enriched_social_quality` from Notion journal enrichment.

### Feature #18 — Automated Threshold Learning (Anomaly Detector v2.0)
- **Adaptive Z thresholds** via coefficient of variation (CV):
  - Low-CV metrics (<0.15, e.g. RHR, weight): Z=1.5 SD (tight — small moves matter)
  - Medium-CV (0.15-0.3): Z=1.75 SD
  - High-CV (>0.3, e.g. steps, tasks): Z=2.0 SD (naturally variable)
- **Day-of-week normalization** for steps, tasks_completed, completion_pct — separate weekday/weekend baselines
- **Minimum absolute change filters**: weight ±1.5 lbs, steps ±2000, RHR ±3 bpm
- **Updated metrics**: chronicling→habitify, added Body Battery + Garmin Stress (13 metrics across 7 sources, up from 11/6)
- **Threshold transparency**: cv, z_threshold, baseline_type, sample_size stored in each anomaly record
- `detector_version: "2.0.0"` field added to DynamoDB records
- Lambda: `anomaly-detector` (handler updated to `lambda_function.lambda_handler`)

## Remaining Work for Feature #21 (Insight Email Parser)

The Lambda, IAM role, CloudWatch alarm, and S3 invoke permission are all live. To activate inbound email flow:

1. **DNS MX record** for `mattsusername.com`:
   ```
   MX 10 inbound-smtp.us-west-2.amazonaws.com
   ```
   (Or use a subdomain like `reply.mattsusername.com`)

2. **SES Receipt Rule Set** (one-time):
   ```bash
   aws ses create-receipt-rule-set --rule-set-name life-platform-inbound --region us-west-2
   aws ses set-active-receipt-rule-set --rule-set-name life-platform-inbound --region us-west-2
   ```

3. **SES Receipt Rule:**
   ```bash
   aws ses create-receipt-rule \
     --rule-set-name life-platform-inbound \
     --rule '{"Name":"insight-reply-rule","Enabled":true,"Recipients":["insights@mattsusername.com"],"Actions":[{"S3Action":{"BucketName":"matthew-life-platform","ObjectKeyPrefix":"raw/inbound_email/"}}]}' \
     --region us-west-2
   ```

4. **S3 bucket policy** — allow SES to write to `raw/inbound_email/`

5. **S3 event notification** — trigger Lambda on `raw/inbound_email/` object creation

6. **Update email templates** — add `Reply-To: insights@mattsusername.com` to Daily Brief, Weekly Digest, Monthly Digest, and Anomaly Alert emails
