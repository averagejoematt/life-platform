## What was accomplished this session

### DynamoDB TTL field name fix ✅

The enterprise review (session h) flagged DynamoDB TTL as unconfirmed. This session resolved it fully.

**Diagnosis:**
- `aws dynamodb describe-time-to-live` confirmed TTL is enabled on the `life-platform` table with attribute name `ttl`
- `mcp_server.py` had a field name mismatch: both the write path (`ddb_cache_set`) and read path (`ddb_cache_get`) used `ttl_epoch` instead of `ttl`
- Effect: DynamoDB's TTL engine never saw the expiry timestamp — cache entries would persist indefinitely if the nightly warmer failed to overwrite them

**Fix applied (mcp_server.py):**
```python
# ddb_cache_set() — line ~220
# BEFORE:
"ttl_epoch": Decimal(str(ttl_epoch)),

# AFTER:
"ttl": Decimal(str(ttl_epoch)),  # DynamoDB TTL attribute — auto-expires after 26h

# ddb_cache_get() — line ~162
# BEFORE:
ttl = item.get("ttl_epoch")

# AFTER:
ttl = item.get("ttl")
```
