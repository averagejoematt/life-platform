## What Was Done This Session

### Task 10: Digest utility consolidation — COMPLETE (pending deploy)

**New file created:** `lambdas/digest_utils.py`

A shared pure-Python module (~270 lines) containing all utilities that were
previously duplicated (sometimes divergently) between `weekly_digest_lambda.py`
and `monthly_digest_lambda.py`.

**Contents of digest_utils.py:**
- Pure scalar helpers: `d2f`, `avg`, `fmt`, `fmt_num`, `safe_float`
- `dedup_activities` — Garmin→Strava duplicate removal (15-min window, richness-based)
- `_normalize_whoop_sleep` — canonical Whoop field alias mapping (v2.55.0 SOT)
- List-based extractors: `ex_whoop_from_list`, `ex_whoop_sleep_from_list`, `ex_withings_from_list`
- Banister adapters: `compute_banister_from_list`, `compute_banister_from_dict`, `_banister_core`

**weekly_digest_lambda.py:**
- Pure refactor — removed ~80 lines of local function defs
- Now imports from `digest_utils`
- No behaviour change

**monthly_digest_lambda.py — v1.2.0 (5 bug fixes):**

| Bug | Was | Now |
|---|---|---|
| `ex_macrofactor` field names | `calories`, `protein_g` (wrong — returning zero data silently) | `total_calories_kcal`, `total_protein_g` |
| `ex_macrofactor` targets | Hardcoded 180g protein / 1800 cal constants | Profile-driven from PROFILE#v1 |
| Profile SK | `PROFILE` (wrong — silent fallback to defaults) | `PROFILE#v1` |
| `ex_strava` dedup | Not called (counting Garmin→Strava duplicates) | `dedup_activities()` applied per day |
| Banister | Local `compute_banister()` without dedup | `compute_banister_from_list()` with dedup |

Additional improvements:
- Zone 2 HR range is now profile-driven (`max_heart_rate * 60%–70%`); falls back to constants
- `fetch_range()` now applies `d2f()` inline (was returning raw Decimal objects)
- `compute_annual_goals()` accepts profile dict — eliminates redundant DynamoDB fetch
- Zone 2 label in `build_html` shows actual computed HR range (not hardcoded constants)
