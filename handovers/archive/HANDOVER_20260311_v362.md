### v3.6.4 — anomaly_detector v2.4.0 HRV log-transform
- Added `LOG_TRANSFORM_METRICS = {"hrv"}` constant — named and extensible
- `compute_baseline()` now accepts `log_transform` param: mean/SD computed on log(values) in log domain; CV computed in original domain to keep adaptive-threshold logic consistent across metrics
- `check_anomalies()` applies `log()` to HRV observation before Z computation; display values back-converted to original ms units via delta method
- Flagged dict gains `log_transform` (bool) and `distribution_note` ("lognormal_z"/"gaussian_approx") fields for DynamoDB transparency
- **Effect:** reduces false high-HRV flags; makes low-HRV Z-scores more precise. HRV=78ms: raw Z=+3.4 → log Z=+2.8. HRV=28ms: raw Z=−2.7 → log Z=−3.4 (more sensitive to genuine lows)
- Corrected a doc error introduced in v3.6.3: DoW σ was already correctly computed on day-type-filtered values — the claim it was mixed was wrong
- Syntax validated, smoke-tested, deployed via `git push` → GitHub Actions
