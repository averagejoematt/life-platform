## Next session: QA / Code Quality Deep Dive

The next session should be a thorough quality assurance pass across the codebase. Priority areas:

### A. Calculated fields & formulas to audit
- `weight_projection()` ‚Äî fix the flat-window bug above; review rate calculation math
- `ex_macrofactor()` ‚Äî calorie/protein hit rate divides by `days` (total records) not `len(cals)` ‚Äî if any records are missing `total_calories_kcal` this understates hit rate; decide if intentional
- `compute_banister()` ‚Äî verify CTL/ATL/TSB math against canonical Banister model; check edge cases with sparse training data
- `trend4()` ‚Äî currently uses first vs last value; consider whether slope should be linear regression across all 4 points
- `score_badge()` nutrition threshold ‚Äî currently uses avg of calorie_hit_rate + protein_hit_rate; review weighting logic
- `compute_sleep_debt()` ‚Äî verify rolling window and target comparison

### B. DynamoDB query patterns to audit
- All `fetch_dates()` calls ‚Äî `get_item` per day (N calls per week window). For 7 days = 7 round trips per source √ó 8 sources = 56 DynamoDB calls at digest time. Consider batching with `batch_get_item`
- Date window off-by-one risk: `dates_back(1, 7)` generates today-1 through today-7. Verify this always captures Mon‚ÄìSun for a Sunday-morning run
- `fetch_stale_insights()` ‚Äî does a full partition scan with filter; acceptable now (<100 insights ever), but no secondary index. Flag for future if insight volume grows
- Banister `fetch_range()` ‚Äî 60-day window; confirm boundary dates are correct relative to today

### C. Code quality / efficiency
- Error handling: most extractors silently swallow exceptions (`except Exception: pass`). Consider logging failures rather than silently returning empty
- `d2f()` (Decimal ‚Üí float) is only called at `build_html` time (line 539), not before extractors run. Extractors do `float(r["total_calories_kcal"])` inline ‚Äî works, but inconsistent pattern
- `ex_macrofactor` does `float()` cast explicitly; other extractors may not ‚Äî audit for Decimal handling consistency
- Lambda cold-start: MCP server loads all tools on init ‚Äî review if any expensive operations run at module level
- Dead code: `backfill_macrofactor_workouts.py` and `migrate_hevy_to_macrofactor.py` may be obsolete ‚Äî confirm and remove or archive
- Weekly digest is 882 lines ‚Äî consider splitting into `digest_data.py` (fetch/extract), `digest_email.py` (HTML build), `digest_main.py` (orchestration)

### D. Smoke test checklist
