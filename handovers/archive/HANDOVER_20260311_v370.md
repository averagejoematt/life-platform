## What Happened

MCP Lambda (`life-platform-mcp`) was in a crash loop. Root cause: `from mcp.tools_longevity import *` in `registry.py` — the file was never created. Fixing that revealed 3 more stacked bugs (broken imports in tools_memory, tools_decisions, tools_hypotheses; plus an unimplemented function reference for defense_patterns).

## What We Did

### 1. Fixed 4 Crash Bugs
- Removed `tools_longevity` import (module never existed)
- Shimmed `tools_memory.py` imports (`_get_table`/`_get_user_id` → `mcp.config`)
- Shimmed `tools_decisions.py` imports (same pattern)
- Shimmed `tools_hypotheses.py` import (`mcp.db` → `mcp.config.table`)
- Removed `get_defense_patterns` registration (function never written)

### 2. Consolidated 148 → 115 Tools
Full Technical Board review conducted. Removed:
- 4 never-implemented tools (longevity × 3, defense_patterns × 1)
- 13 domain-specific correlation tools (`get_cross_source_correlation` covers all)
- 8 Claude-derivable tools (daily_summary + reasoning)
- 6 low-usage/aspirational tools
- 1 labs tool (Claude derives from lab data)
- 2 BoD admin tools (→ admin scripts)

### 3. Built Registry Integrity Test
`tests/test_mcp_registry.py` — 7 checks (R1-R7), all offline, 0.33s:
- R1: imports resolve to real files
- R2: every `"fn"` value exists as a defined function
- R3: schema structure valid
- R4: no duplicate tool names
- R5: tool count in expected range (100-130)
- R6: registry.py syntax valid
- R7: all tools_*.py modules parseable

Wired into `deploy/deploy_mcp_consolidation.sh` — deploy aborts if lint fails.
