### Nightly Activity Enrichment Lambda — built and deployed
- New Lambda: `activity-enrichment`
- EventBridge: 06:00 UTC (10pm PT) nightly — runs after all daily source syncs complete
- Writes `enriched_name` and `enriched_at` fields to each activity in DynamoDB without overwriting source data
- No drift risk — original `name` field untouched, enrichment is additive

**Enriched name format:**
```
{activity_name} — {city}, {state} · {dist}mi · {elev}ft · {hr}bpm avg · {recovery_emoji} · {percentile_note} · {PR count}
```
- Location prepended only for generic names (Morning Run, Afternoon Hike, etc.)
- Recovery emoji (🔴🟡🟢) pulled from Whoop recovery score that day
- Percentile note only shown if top 10% all-time elevation or distance (not noise)
- Each component omitted gracefully if data missing

**Backfill result:** 2,636 activities enriched across 1,099 days

**Known behaviour:**
- WHOOP auto-sync duplicates (same session from WHOOP + Hevy) show as two records — expected, not a bug
- Hevy activities with emojis in name (e.g. `Morning workout ☀️`) pass through unchanged — emoji breaks generic detection, acceptable

## Things to watch / known issues
- Location fields on historical Strava records will be null — only new activities get them
- Enrichment Lambda backfill can be re-run safely (idempotent — only writes if enriched_name changed)
- To re-backfill: `aws lambda invoke --function-name activity-enrichment --payload '{"backfill":true,"start_date":"2020-01-01"}' --cli-binary-format raw-in-base64-out --region us-west-2 /tmp/out.json`
