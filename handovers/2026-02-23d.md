### 2. Anomaly Detector Lambda — built and deployed ✅
- **File:** `anomaly_detector_lambda.py`
- **Function:** `anomaly-detector` (us-west-2, python3.11, 90s, 256MB)
- **Schedule:** `cron(5 15 * * ? *)` → 8:05am PT daily — fires BEFORE daily-brief
- **IAM role:** `lambda-anomaly-detector-role` (new, dedicated, least-privilege: DynamoDB GetItem/Query/PutItem + Secrets Manager anthropic key + SESv2 SendEmail)

**What it does:**
- Fetches yesterday's values for 9 metrics across 6 sources
- Computes 30-day rolling mean + SD for each metric (minimum 7 days history required)
- Flags metrics >1.5 SD from baseline, directional (low_is_bad logic per metric)
- Multi-source rule: only alerts when 2+ distinct sources are flagged
- If multi-source: calls Haiku for a root cause hypothesis, sends alert email
- Always writes a DynamoDB anomaly record (severity: none/moderate/high) regardless

**Metrics monitored:**
| Source | Field | Direction |
|--------|-------|-----------|
| Whoop | recovery_score | Low is bad |
| Whoop | hrv | Low is bad |
| Whoop | resting_heart_rate | High is bad |
| Eight Sleep | sleep_score | Low is bad |
| Eight Sleep | sleep_efficiency | Low is bad |
| Withings | weight_lbs | Both directions |
| Apple Health | steps | Low is bad |
| Todoist | tasks_completed | Low is bad |
| Chronicling | total_score | Low is bad |

**Test result:** `{"flagged_count": 0, "multi_source": false, "severity": "none", "alert_sent": false}` — clean day, working correctly.
