## Decisions Made This Session

### Garmin as biometrics-only source (not activities)
Activities route Garmin → Strava automatically. Garmin Lambda pulls biometrics only:
- Body Battery (proprietary energy reserve, 0-100 — nothing else in platform has this)
- Physiological stress (continuous HRV-derived, objective — Exist.io will add subjective complement)
- Overnight HRV (cross-check with Whoop)
- RHR (cross-check with Whoop)
- Respiration rate (cross-check with Eight Sleep)
- Steps (cross-check with Apple Health)

### garminconnect Python library (not raw HTTP)
Garmin SSO is complex (CSRF tokens, MFA flows). `garminconnect` + `garth` libraries handle auth cleanly. Library must be bundled in Lambda zip. `deploy_garmin.sh` handles this automatically via `pip install --target`.

### `setup_garmin_auth.py` for first auth
Garmin may require MFA on first login — this must run interactively on local machine. Saves `garth` OAuth tokens to Secrets Manager `life-platform/garmin`. Lambda loads stored tokens; falls back to password re-login if expired.

### Readiness score rebalanced to 5 components
- Whoop recovery:     40% → 35%
- Eight Sleep score:  30% → 25%
- HRV 7-day trend:    20% (unchanged)
- TSB training form:  10% (unchanged)
- Garmin Body Battery: 0% → 10% (new)

Device agreement section added to `get_readiness_score` output — Whoop vs Garmin HRV/RHR delta with confidence rating (high/moderate/low). Flags when score may be less reliable.
