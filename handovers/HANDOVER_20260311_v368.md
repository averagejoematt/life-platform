## Linter maintenance guide
When adding a new Lambda:
1. Add to `ALL_LAMBDAS` in `test_wiring_coverage.py`
2. Add to appropriate category list (INGESTION_LAMBDAS / AI_OUTPUT_LAMBDAS / PROMPT_LAMBDAS)
3. Add to `DDB_LAMBDAS` + appropriate subset in `test_ddb_patterns.py`
4. Add to `lambda_map.json` (handler consistency check covers it automatically)
5. If it has a known gap, add to the relevant `_KNOWN_GAPS` set with a comment

When retiring a secret:
1. Add to `ci/deprecated_secrets.txt` with deadline and replacement noted
2. Run deprecated secrets scan locally: `grep -rn "secret-name" lambdas/ mcp/ cdk/`
3. Remove entry after all refs are cleared AND secret is confirmed deleted in AWS
