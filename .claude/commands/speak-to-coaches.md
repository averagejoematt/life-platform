Work the open coach check-in queue conversationally, one question at a time, and record
Matthew's answers verbatim. See `docs/coaching/CHAT_MODES.md` for the route-the-takeaways
contract and the verbatim/skip rules this command follows exactly.

## Arguments: $ARGUMENTS

Optional: a `coach_id` (sleep, nutrition, training, mind, physical, glucose, labs,
explorer) to hear from one specific coach. Empty: let the queue auto-pick from the most
informative signal.

## Instructions

### 1. Pull the queue

Call `get_coach_checkin_queue` (pass `coach_id` if `$ARGUMENTS` gave one). This returns
up to 3 open qualitative questions from Matthew's AI coaches — questions whose verbatim
answers pair with (or explain the absence of) the quantitative data. If the queue is
empty, say so plainly and stop; don't manufacture a question. Fresh questions are only
generated by the tool itself when the queue is empty — this command never invents one.

### 2. Ask ONE question at a time, conversationally

Don't dump all 3 at once. Present the first question in your own words if it reads
stiffly, but don't change its substance. Give Matthew room to answer naturally — follow
up once if his answer is genuinely ambiguous, but don't interrogate.

**Skipping is always valid, zero penalty, never contested.** If Matthew doesn't want to
answer one, move on immediately — don't ask why, don't come back to it later in the same
session.

### 3. Log the answer VERBATIM

Once Matthew has responded (or declined), call `log_coach_checkin`:
- `checkin_id`: from the queue item (starts `CHECKIN#`).
- `coach_id`: pass it if known — speeds the lookup.
- `answer`: Matthew's actual words. **Do not paraphrase, tidy, or summarize** — ADR-104
  is explicit that this is his voice, not a Claude gloss. If he rambled, the verbatim
  text can be trimmed of pure filler, but the substance and phrasing stay his.
- `skip`: `true` if he declined, and omit `answer` in that case.
- `tags`: optional, up to 5, only if an obvious topic tag helps future retrieval.

Repeat steps 2-3 for each remaining question in the queue Matthew is willing to work
through — stop whenever he wants to stop, the queue persists across sessions.

### 4. Optional: habit-reflection pass

Only if Matthew is already in a reflecting mood after the check-in questions (don't
pivot into this if the check-ins felt like enough) — call `get_habit_reflection_queue`
(default `days=7` for a weekly view). Pick one or two items, ask about them naturally
("missed meditation on the 14th — anything going on that day?"), then call
`log_habit_reflection` with whatever he says (`trigger`, `reward`, `why_missed`, or
free-text `context` — pass whichever fields fit, none are all required beyond `habit`).
Never nag or schedule this pass; it exists only to make the ask possible when the moment
is right.

### 5. Optional: weekly Field Notes response

If this is a weekly session (or Matthew brings up the AI Lab Notes), call
`get_field_notes` for the current week (or a specific `week`, e.g. `'2026-W14'`). Read
back or summarize the AI's present/lookback/focus paragraphs, ask what he agrees or
disagrees with and what it missed, then call `log_field_note_response` with `week`,
`notes` (his prose response), and optionally `agreement` (`agree`/`disagree`/`mixed`),
`disputed` (specific claims he pushes back on), and `added` (what he noticed that the AI
missed). This only works if that week's AI Lab Notes already exist — if `get_field_notes`
comes back empty, say so and skip this step rather than inventing a response target.

### 6. Close out

Briefly summarize what got logged this session (which questions answered/skipped, any
habit reflections, any Field Notes response) so Matthew has a clean sense of what's now
durable vs. what's still open in the queue for next time.
