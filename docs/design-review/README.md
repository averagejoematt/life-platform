# Design-review kit — paste-ready expert-panel prompts (averagejoematt.com)

Each file is a **self-contained, two-stage prompt** for a claude.ai design review. Open the page's
file, copy it whole, paste it into your Project chat, attach the screenshots — that's it.

**Two-stage by design:** Stage 1 has the model **assemble the ideal expert panel for that page's
domain itself** (a nutrition page convenes different specialists than a strength page or a
sleep-science page — the model picks who's in the room and justifies it). Stage 2 convenes them
and runs the review. Each prompt embeds the platform spirit, the design system, and **your real
data inventory** for that page (the part only Claude Code can supply accurately), so the
self-assembled panel reasons about what actually exists — not what it imagines.

## The loop
1. Pick a page → open its file here → copy the whole prompt.
2. Capture screenshots of that page (see below) → paste prompt + screenshots into the Project.
3. Get the expert panel's uplevel (visuals, features, what-to-cut, what-data-to-capture).
4. **Hand the ideas back to Claude Code** — it builds them + reality-checks against the real data
   (flagging anything premature at week one, like the QA passes did).

## Screenshots — how to capture each page well
For each page, attach:
- **Full-page desktop scroll** (the whole page, not just the fold) — a full-page-capture browser
  extension is easiest for the long Evidence pages.
- **Mobile width** (resize to ~390px or use device mode).
- **Both light + dark** only if the page's look differs meaningfully.
- The Cockpit (`/now/`) and each Evidence sub-page as their own shots.

claude.ai analyzes pasted images well but **cannot browse your live site** — screenshots are the
reliable way to give it the visuals. Each prompt also references the live data so the panel sees
what's behind the pixels.

## Structure of every prompt (identical except the DATA block)
1. **PLATFORM CONTEXT** + **DESIGN SYSTEM** — the spirit + constraints (same in every file).
2. **STAGE 1 — assemble the room** — the model designs the page's ideal specialist panel + a
   data-viz designer, and justifies each pick for *this* domain.
3. **THE PAGE + ITS REAL DATA** — the page's actual data inventory (the only block that changes
   file-to-file).
4. **STAGE 2 — the review task** — one story / specialist visuals / features / what-to-cut /
   what-data-to-capture, ranked, with premature-at-week-one flags.

To review a page not covered here, copy any file and swap only the **DATA** block (ask Claude Code
to dump that page's real `/api` fields).
