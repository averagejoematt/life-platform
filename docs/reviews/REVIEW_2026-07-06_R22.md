## ⚠️ CRITICAL — MCP remote-auth hardening (handled out-of-band; details held privately)

R22 found a **live, high-severity gap in the MCP remote-auth boundary** affecting the private
health-data surface. Because this repo is **public**, the reproduction detail is deliberately
**not recorded here or in any tracked file** — it lives in the operator's private memory
(`security-r22-mcp-token-exposure`). Public tracking is the two terse hardening issues under
epic **#722** (SEC-01: harden the `/token` / `/authorize` remote-auth flow; SEC-02: rotate + stop
committing the Function URL). Both are being remediated out-of-band ahead of any public disclosure.

Two lower-risk, defensively-scoped security items are filed normally: **SEC-03** (wire the inert
site-api origin-header guard) and **SEC-04** (delimit untrusted reader text on board_ask).
